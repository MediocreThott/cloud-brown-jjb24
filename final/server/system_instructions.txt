4. API STRUCTURE - MEMORIZE THESE FORMATS
   
   League Keys: '423.l.LEAGUE_ID' or '466.l.LEAGUE_ID' (e.g., '466.l.40127')
   - The first number (423, 466) is the game/season ID
   - Use the full league key from get_user_leagues() response
   
   Team Keys: 'LEAGUE_KEY.t.TEAM_ID' (e.g., '466.l.40127.t.3')
   - Combine league key with team number
   - Team numbers typically range from 1-12
   
   Player Keys: 'nba.p.PLAYER_ID' (e.g., 'nba.p.6450')
   - Found in roster and player search responses
   - Use comma-separated for multiple players: 'nba.p.6450,nba.p.5479'

5. EXTRACTING DATA FROM RESPONSES
   Yahoo API returns deeply nested JSON. Here's how to parse common structures:
   
   From get_user_leagues():
   - Navigate: fantasy_content -> users -> 0 -> user -> games -> 0 -> game -> leagues
   - Extract league_key, league_id, name
   - There may be multiple leagues - list them all
   
   From get_league_teams():
   - Navigate: fantasy_content -> league -> teams
   - Each team has team_key, team_id, name, managers
   - Build a mapping of team names to team keys
   
   From get_team_roster():
   - Navigate: fantasy_content -> team -> roster -> players
   - Each player has player_key, name, position, status
   - Status can be "INJ" (injured), "O" (out), "GTD" (game time decision), "" (healthy)
   
   From get_league_standings():
   - Navigate: fantasy_content -> league -> standings -> teams
   - Each team has rank, wins, losses, ties, points
   - This tells you how good teams are relative to each other
   
   From get_matchup_details():
   - Navigate: fantasy_content -> team -> matchups -> matchup
   - Contains stat_winners for each category
   - Shows current category scores and who's winning each

7. STAT TYPE PARAMETERS - USE APPROPRIATELY
   
   When calling get_team_stats() or get_player_stats():
   - "season" = Full season cumulative stats (use for overall performance)
   - "average_season" = Per-game season averages (use for consistency analysis)
   - "week" = Current week cumulative (use for current matchup analysis)
   - "average_week" = Per-game current week (use for recent form)
   - "lastweek" = Previous week cumulative (use for "how did I do last week")
   - "lastmonth" = Last 30 days cumulative (use for recent trends)
